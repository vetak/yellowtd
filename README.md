# YellowTD

Браузерная tower defense в пустынном сеттинге, вдохновлённая GreenTD из Warcraft 3.

**Играть онлайн:** https://vetak.github.io/yellowtd/

Версия **1.5.0**: прогрессия карт — карты открываются последовательно по мере
прохождения (Классика → Каньон → Пустоши), заблокированные показаны с замком и
условием открытия. Новая endgame-карта **«Пустоши»**: длинный шестиполосный
серпантин, полный арсенал из 8 башен (все семь классических + грозовая цепная),
24 самых крепких волны и два босса — Джаггернаут и Бегемот
+ в подсказке значков будущих волн («Впереди») статусы теперь
расшифрованы словами в скобках — «5. Пыльный рой (воздух)», «(босс)», «(иммун)»
и т.д. (при наведении мышью); а на телефоне тап по значку волны показывает её
полное описание — состав, статус и бонус
+ мобильная полировка — долгое нажатие на поле показывает инфо
о враге или башне под пальцем (на тач нет наведения мышью), плюс оптимизация:
панель волны и полоса будущих волн перестраиваются только при смене волны, а не
каждый кадр
+ адаптивная вёрстка и тач-управление — на телефоне и планшете
интерфейс раскладывается в колонку, поле масштабируется под ширину экрана,
башни строятся тапом, крупнее кнопки под палец, убран двойной тап-зум по полю;
кнопки +/− приближают поле (панорама пальцем) для точной установки башен;
в горизонтальной ориентации — раскладка «поле + панель сбоку», карта целиком
вписана по высоте экрана
+ объёмная 2D-подача — текстурированный песок с дюнами,
дорога с колеёй и тенью обочины, тени и шейдинг у башен и врагов, частицы
боя (всё рисуется кодом, без файлов-ассетов) + «Темп волн»: следующую волну
можно отправить досрочно, поверх текущей, за убывающий бонус золота (до двух
волн одновременно в поле) + подсказка для новых игроков, переходы экранов,
боевая обратная связь + режим «Эндлесс» (бесконечные волны без победы) и
испытания-модификаторы (без продажи, одна башня на тип, половина золота,
ускоренные крипы) + сложность «Кошмар» и таблица рекордов + звук и музыка
на WebAudio (синтез в рантайме, без файлов) + три карты, открываются
последовательно (прогрессия):
- **«Классика»** (открыта сразу) — пустынный серпантин, 7 типов башен, 36 волн,
  два босса (волны 24 и 36) и два героя-мини-босса (волны 12 и 33);
- **«Каньон»** (после Классики) — узкое ущелье с 14 поворотами, 6 башен
  (включая «Башню бури» с цепной молнией), 24 волны, два босса и герой;
- **«Пустоши»** (после Каньона) — endgame: длинный шестиполосный серпантин,
  полный арсенал из 8 башен, 24 самых крепких волны и два босса.

Спец-волны (иммунитет, реген, герои, бонусные), три сложности,
у каждой версии карты своё автосохранение, экспорт/импорт сейвов.

Управление на ПК: мышь, клавиши 1–7 — башни, пробел — пауза, Esc — меню.
Shift/Ctrl+клик — серийное строительство, Ctrl+клик по «Улучшить» — улучшить все.

Управление на телефоне/планшете: тап по кнопке башни, затем тап по клетке —
строительство; тап по башне — улучшение/продажа; долгое нажатие на поле —
инфо о враге/башне; кнопки +/− у поля — приблизить и панорамировать пальцем.
В горизонтальной ориентации карта видна целиком, панель управления — сбоку.

## Запуск локально

Игра — чистая статика без сборки и внешних зависимостей (весь код и графика
генерируются в рантайме, ассетов-файлов нет). Запустить можно двумя способами:

- **Просто открыть `index.html`** двойным кликом — работает по `file://`, так как
  все скрипты подключаются относительными путями, а сеть не используется.
- **Через локальный сервер** (удобно для мобильной проверки в той же сети):
  `python3 -m http.server 8000`, затем открыть `http://localhost:8000/`.

Деплой: `main` автоматически публикуется на GitHub Pages
(https://vetak.github.io/yellowtd/).

## Тесты

Нужен только Node.js (без npm-зависимостей):

- `node test/smoke.js` — движок и баланс (детерминированная симуляция, победа на
  обеих картах, механики башен, спец-волны, эндлесс, модификаторы, «Темп волн»).
- `node test/dom-smoke.js` — загрузка, меню, HUD, ввод (вкл. тач: зум, долгое
  нажатие, тап по значкам волн), сохранения, рекорды.

## Структура и как расширять

Код разложен по папкам, у каждой свой `README.md`:

- `engine/` — детерминированная симуляция (без DOM и `Math.random`);
- `render/` — отрисовка (только читает состояние);
- `audio/` — звук/музыка на WebAudio (только читает события);
- `ui/` — HUD, ввод, меню, сохранения;
- `data/` — все конфиги: правила, версии карт, башни, крипы, волны.

Добавление контента (новая карта/башня/крип/волна) — data-driven, см.
`data/README.md`.
