# audio

Папка отвечает за звук.

## Что здесь есть
- `audio.js` — `SoundEngine`: синтез звуковых эффектов и процедурной музыки
  на WebAudio, без файлов-ассетов. Читает события симуляции (как `render/`)
  и озвучивает их; ничего не пишет обратно в состояние игры.

## Что можно менять
- Звуковую палитру (тембр, огибающие, ноты музыки).
- Маппинг событие → звук.
- Громкость по умолчанию.

## Что важно помнить
- `engine` ничего не знает про звук — `audio` только читает `sim.state.events`.
- Никаких файлов-ассетов: всё синтезируется в рантайме, чтобы file:// и
  офлайн desktop-сборка работали без внешних запросов.
- `AudioContext` можно создавать только после жеста пользователя — не
  пытаться проигрывать звук до `unlock()`.
- Headless-окружение (тесты, SSR) не имеет `AudioContext` — модуль должен
  молча деградировать, а не бросать исключения.
