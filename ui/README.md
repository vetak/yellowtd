# ui

Папка отвечает за интерфейс и ввод.

## Что здесь есть
- `menu.js` — меню (главное, пауза, выбор версии карты с превью маршрутов,
  сложность, настройки).
- `ui.js` — HUD, панель башен, подсказки, обработка действий.
- `storage.js` — сохранения (слот на каждую версию карты:
  `yellowtd.save.<versionId>`) и пользовательские настройки.
- `platform.js` — единственное место, знающее про Tauri (выход из приложения,
  файловые диалоги экспорта/импорта); в браузере деградирует в скачивание/загрузку.

## Что можно менять
- Кнопки.
- Подсказки.
- Панель информации.
- Управление и хоткеи.

## Мобильное управление (1.3.0)
- **Раскладка адаптивная и живёт в `style.css`** (медиазапросы), а не в JS.
  Ниже 1024px интерфейс раскладывается в колонку (портрет) или «поле + панель
  сбоку» (ландшафт, `max-height: 600px`). Поле вписывается по ширине/высоте
  экрана через `max-width`/`max-height` при `width/height: auto` (contain).
- **Зум поля** — кнопки `#zoom-in/#zoom-out/#zoom-fit`. JS (`_setZoom`) хранит
  масштаб 1..3 и пишет его в CSS-переменную `--board-zoom` на `#board-scroll`;
  CSS множит на неё размер canvas, `#board-scroll` прокручивается (панорама).
  На десктопе переменная не используется — код там no-op.
- **Точность тапа не зависит от масштаба:** координаты пересчитываются через
  `getBoundingClientRect` (`_canvasPos`), поэтому попадание в клетку верное при
  любом зуме и CSS-масштабе.
- **`_revealSelectedTower`** на мобиле подскролливает панель выбранной башни в
  зону видимости (в портрете она под полем). Определение мобильной раскладки —
  `_isMobileLayout()` через `matchMedia`; в headless-тестах деградирует в no-op.

## Что важно помнить
UI вызывает команды симуляции, но не должен дублировать боевую логику из `engine`.
Тексты подсказок нейтральны к платформе: и тап, и клавиши/мышь (клавиши в скобках).
