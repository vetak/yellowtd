# data

В этой папке лежат игровые конфиги.

## Что здесь есть
- `config.js` — общие правила: GAME_VERSION, экономика, сложности, настройки.
- `versions/<id>/` — версии карты, у каждой свой полный набор:
  - `map.js` — размеры карты и путь;
  - `towers.js` — башни, уровни, стоимость, дальность, урон;
  - `creeps.js` — базовые типы крипов;
  - `waves.js` — состав волн.
- `versions.js` — реестр версий (`VersionsConfig`, `VersionOrder`):
  id, название, описание и ссылки на конфиги версии.

Файлы версий наполняют namespace-объекты (`ClassicVersion`, `CanyonVersion`)
и подключаются через `<script>` до `versions.js` — file:// работает без
сборщиков и динамической загрузки.

## Что можно менять
- Баланс.
- Названия.
- Количество волн.
- Геометрию карты.
- Добавлять новые версии: папка `versions/<id>/` + запись в `versions.js`
  (и планы застройки в `test/smoke.js`).

## Что менять осторожно
- Формат полей конфигов, потому что на них завязаны `engine` и `ui`.
- При изменении карты/волн обновляй планы застройки в `test/smoke.js`.
