# data

В этой папке лежат игровые конфиги.

## Что здесь есть
- `config.js` — общие правила: GAME_VERSION, экономика, сложности, настройки.
- `versions/<id>/` — версии карты, у каждой свой полный набор:
  - `map.js` — размеры карты и путь;
  - `towers.js` — башни, уровни, стоимость, дальность, урон;
  - `creeps.js` — базовые типы крипов;
  - `waves.js` — состав волн.
- `versions.js` — реестр версий (`VersionsConfig`, `VersionOrder`):
  id, название, описание и ссылки на конфиги версии.

Файлы версий наполняют namespace-объекты (`ClassicVersion`, `CanyonVersion`)
и подключаются через `<script>` до `versions.js` — file:// работает без
сборщиков и динамической загрузки.

## Что можно менять
- Баланс.
- Названия.
- Количество волн.
- Геометрию карты.
- Добавлять новые версии: папка `versions/<id>/` + запись в `versions.js`
  (и планы застройки в `test/smoke.js`).

## Как добавить контент (data-driven)

**Новую башню** — добавь запись в `versions/<id>/towers.js`:
- `name`, `color`, `role`, `hotkey`, `targets` (`['ground']`, `['air']` или оба);
- `levels[]` — по уровню: `cost`, `damage`, `range`, `cooldown` и, при желании,
  один из спец-эффектов (см. каталог ниже). UI сам добавит кнопку и хоткей.

**Нового крипа** — запись в `versions/<id>/creeps.js`: `name`, `type`
(`ground`/`air`), `speed`, `radius`, `livesCost`, `color`; для боссов —
`boss: true` (+ `slowCap`). Затем ссылайся на него в группах волн по ключу.

**Новую волну** — элемент массива в `versions/<id>/waves.js`: `name`, `bonus`,
`groups[]` (`creep`, `count`, `interval`, `hp`, `bounty`, опц. `armor`) и любой
из флагов спец-волн: `boss`, `hero`, `immuneToSlow`, `regen` (hp/сек),
`extra` (бонусная: побеги не стоят жизней, награда ×2).

**Новую версию карты** — папка `versions/<id>/` со всеми четырьмя файлами +
запись в `versions.js`; обнови планы застройки в `test/smoke.js`.

### Каталог эффектов башен, которые движок уже понимает
Эти поля уровня работают **без правок `engine`** (см. `engine/sim.js`):
`splash` (радиус урона по площади), `slowFactor`+`slowDuration` (замедление),
`poisonDps`+`poisonDuration` (яд, игнорирует броню), `shots` (залп по N целям),
`chain`+`chainRange` (цепная молния). Любой **новый** тип эффекта потребует
поддержки в `engine/sim.js` (и теста в `test/smoke.js`).

## Что менять осторожно
- Формат полей конфигов, потому что на них завязаны `engine` и `ui`.
- При изменении карты/волн обновляй планы застройки в `test/smoke.js`.
- Изменение `GAME_VERSION` делает старые сейвы несовместимыми (проверка версии
  в `ui/storage.js`) — ожидаемо для релиза, но помни об этом.
